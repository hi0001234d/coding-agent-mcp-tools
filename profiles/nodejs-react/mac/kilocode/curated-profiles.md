# ⚡ Curated Profiles — Node.js + React/Next.js

Pre-configured workflows for the most common real-world scenarios.
Each profile builds on the base setup from [macos-setup.md](./macos-setup.md).

---

## 🧠 AI Amnesia Fix
**Best for**: Developers returning to projects after time away, or anyone hitting "agent keeps forgetting" frustration.

Sets up persistent memory so your agent remembers architectural decisions, known bugs,
and project conventions across every session.

👉 [Stop AI Amnesia → project-memory.md](./project-memory.md)

---

## 🐛 Day 2 Debug Automation
**Best for**: Vibe-coded apps that are working but need maintenance, refactoring, or debugging.

Automated Semgrep scanning + Language Server navigation + code-review checklist
so the agent catches what it introduces before you do.

👉 [Fix the Day 2 Problem → debug-automation.md](./debug-automation.md)

---

## 💡 Token Optimization
**Best for**: Large Next.js codebases where the agent hits context limits.

Workflow patterns that extract maximum value from the 4 MCP tools without
dumping entire files into context.

👉 [Optimize Token Usage → optimize-tokens.md](./optimize-tokens.md)

---

## 🔒 Fully Local
**Best for**: Developers working with sensitive codebases, or those without Figma access.

Same 4-tool setup with Figma disabled — everything runs locally, no external API calls
from any MCP tool.

👉 [Fully Local Setup → fully-local.md](./fully-local.md)

---

## 🎧 Customer Support Workflows
**Best for**: Developers building customer-facing features from design specs or support tickets.

Figma Context MCP + Basic Memory workflow for translating designs and requirements
into production-ready Next.js components.

👉 [Customer Support Workflows → customer-support.md](./customer-support.md)
